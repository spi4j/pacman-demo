Bienvenue dans votre nouveau projet de librairie client demo-dsfr-client-rest

Ce fichier vous permet de rapidement prendre en main le démarrage de l'application :

📁 Projets inclus

Quatre projets sont créés :

  • demo-dsfr-client-rest : le projet qui encapsule les trois autres projets, son seul rôle est de contenir le fichier pom.xml parent.
  • demo-dsfr-client-rest-model : le projet de modélisation. Aucune classe Java ne doit être présente dans ce projet. Il n'est, par ailleurs, pas embarqué lors du déploiement de l'application.
  • demo-dsfr-client-rest-server : le projet contenant les classes client d'appel pour les services de type rest.

📂 Emplacement des fichiers de configuration et classe de démarrage

  • Configuration des options pour la génération : demo-dsfr-client-rest-model/pacman-properties/project.properties Utiliser ce fichier de configuration pour modifier les différentes options utilisées pour la génération de code. Ce fichier a initialement été créé suite aux principaux choix dans le formulaire de création mais toute modification sera automatiquement prise en compte lors de la prochaine génération. Ce fichier de configuration interne pour la génération est commun pour l'ensemble des différents types de projet, certaines propriétés ne sont donc pas prises en considération suivant le type de projet"
  • Configuration du nommage des classes et interfaces pour la génération : demo-dsfr-client-rest-model/pacman-properties/nommage.properties Utiliser ce fichier de configuration pour modifier les différentes options de nommage utilisées pour la génération de code. Si par exemple vous désirez que tous la paramètres des méthodes soient préfixés par 'p_', il suffit de modifier la propriété suivante dans le fichier de configuration parameterMethodClassicIn=[CamelCase.LowerFirst/] en parameterMethodClassicIn=p_[CamelCase.LowerFirst/]. Toute modification sera automatiquement prise en compte lors de la prochaine génération.
  • Configuration principale : demo-dsfr-client-rest-server/src/main/resources/application.properties Le fichier de configuration de SpringBoot, avec l'uri de base pour les appels, les timeouts de connexion, etc...

📁 Principaux packages générés

  • demo-dsfr-client-rest-server
    • fr.demo_dsfr_client_rest.app : classes de haut niveau pour le fonctionnement des clients rest.

🚀 Pour commencer rapidement

  1. Si besoin, appliquer la métadonnée @AUTH_TOKEN au niveau de l'opération d'authentification (récupération du jeton)
  2. Générer le code client à partir du fichier de modélisation
  3. Si besoin, compléter la classe Demo_dsfr_client_restTokenTestsHelper avec les données d'authentification.
  4. Effectuer les tests pour vérifier le bon fonctionnement des différents appels.
  5. Procéder au déploiement et à l'intégration de la librairie dans le projet cible.
